# 5.8 Pathway 3: Barlow Twins SSL Pretraining Results

This section reports the Phase 3 Pattern Core results: a TCN+Cross-Attention encoder pretrained with the Barlow Twins SSL objective (Sec. 4.3) on 100K unlabeled anchors and finetuned on 30K labeled anchors per walk-forward split.

## Setup

- **Pretraining**: 100K unlabeled anchors sampled from D1 panel before each split's `train_end` boundary. Encoder TCN+Cross-Attn (119,171 params); projector 3-layer MLP $d \to 128 \to 256$ with batch norm (affine = False). 15 epochs cosine LR schedule on AdamW, lr=3e-4, batch_size=256. Wall-clock: 39.2 min on CPU per pretrain run.
- **Finetuning**: Same encoder, classifier head reinitialized. Differential learning rate (encoder × 0.1, classifier × 1.0) to mitigate catastrophic forgetting. 30K labeled anchors per split (sampled from each split's train window), 5 epochs with early stopping. Wall-clock: 112s per split.
- **Inference**: Same 2000 walk-forward random-sample test anchors per split (consistent with Phase 1 LGBM evaluation in §5.4).

## Per-Split Comparison

| Split | Phase 1: LGBM | Phase 2: TCN (100K) | **Phase 3: BT-SSL + TCN (30K)** |
|---|---|---|---|
| 1 (Q2) RankIC | **+0.177** | +0.084 | +0.078 |
| 1 (Q2) Top-10% | **+3.34%** | +2.48% | +1.55% |
| 2 (Q3) RankIC | −0.015 | +0.034 | **+0.058** |
| 2 (Q3) Top-10% | +0.88% | +0.82% | **+1.72%** |
| 3 (Q4) RankIC | +0.103 | +0.061 | +0.094 |
| 3 (Q4) Top-10% | +0.68% | +1.54% | **+1.62%** |

## Mean Across 3 Splits

| Phase | Labeled Data | RankIC | Top-10% | Top-20% |
|---|---|---|---|---|
| Phase 1: LGBM | 3.5M anchors | **+0.088** | **+1.63%** | +1.45% |
| Phase 2: TCN | 100K (35× less than Phase 1) | +0.060 | +1.61% | +1.45% |
| **Phase 3: BT-SSL + TCN** | **30K labeled + 100K unlabeled** | **+0.077** | **+1.63%** | +1.43% |

**Key finding**: Phase 3 achieves Phase 1's mean Top-10% return (+1.63% vs +1.63%) using only **30K labeled anchors per split** — a **116× reduction in supervised data** relative to Phase 1's 3.5M-anchor training budget. The SSL pretraining "supplies" the missing supervised data via unlabeled invariance learning, validating the data-efficiency hypothesis that motivated the Pathway 3 design (Sec. 7.3).

## Split 2 Bridging — A Concrete Quarterly Win

Of particular interest is Split 2 (2025-Q3), where Phase 1 LGBM fails (RankIC −0.015, marginal Top-10% +0.88%). Phase 3 achieves the *best* result of any single method on this split:

- Phase 1 LGBM: RankIC −0.015, Top-10% +0.88% (failure mode)
- Phase 2 TCN: RankIC +0.034, Top-10% +0.82% (partial bridge)
- **Phase 3 BT-SSL + TCN**: RankIC **+0.058**, Top-10% **+1.72%** (full recovery, +95% over LGBM)

This single-split improvement is consistent with our Section 7.2 hypothesis that LLM agents add value in regimes the LGBM training distribution under-represents. Here, the SSL-pretrained encoder achieves the same effect via redundancy-reduced features that generalize better to the LGBM-failure regime.

## Comparison to LLM Hybrid Approaches

Recall that in §5.4 the best LLM-augmented hybrid router (G_disagreement_boost) achieved Top-10% +1.63% — equal to LGBM but not exceeding it. The Phase 3 BT-SSL approach achieves the same +1.63% Top-10% return at a fraction of the labeled-data cost and **without any LLM inference at deployment time**, making it operationally simpler and dramatically cheaper to deploy ($0 inference cost vs Sonnet 4.6 ~$0.0023 per anchor).

## Summary of Pathway 3 Contribution

The Phase 3 BT-SSL Pattern Core delivers three properties simultaneously:

1. **Match Phase 1 performance** on the deployment-realistic random walk-forward distribution
2. **Bridge Phase 1 failure modes** (Split 2 +95% Top-10% over LGBM)
3. **116× supervised data efficiency**, making future re-training and adaptation to new markets practical

We argue that Pathway 3 represents the strongest deployment-ready upgrade path for the V12-Agentic framework, and recommend it as the default Pattern Core implementation for production migration.
